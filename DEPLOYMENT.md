# Deployment Guide

This guide covers deploying Starlight Labs to production environments.

## Prerequisites

- GitHub account with repository access
- Railway account (or alternative deployment platform)
- Domain name (optional, but recommended)
- SSL certificate

## Railway Deployment

### 1. Connect Repository

1. Log in to [Railway](https://railway.app)
2. Create new project
3. Connect GitHub repository
4. Grant permission to `disputestrike/STARLIGHT-LABS`

### 2. Add Services

#### PostgreSQL Plugin
```bash
# Railway automatically creates DATABASE_URL
# Available in Variables tab
```

#### Redis Plugin
```bash
# Railway automatically creates REDIS_URL
# Available in Variables tab
```

### 3. Set Environment Variables

In Railway dashboard, go to **Variables** and add:

```bash
# Core
NODE_ENV=production
JWT_SECRET=<generate-strong-secret>
ADMIN_SECRET=<generate-strong-secret>

# API Keys
STRIPE_SECRET_KEY=sk_live_...
TWILIO_ACCOUNT_SID=AC...
TWILIO_AUTH_TOKEN=...
RESEND_API_KEY=re_...

# App Configuration
CORS_ORIGIN=https://yourdomain.com
NEXT_PUBLIC_API_URL=https://yourdomain.com

# Database (auto-generated by Railway)
# DATABASE_URL=postgresql://...
# REDIS_URL=redis://...
```

### 4. Deploy

```bash
# Push to main branch
git push origin main

# Railway will automatically:
# 1. Run npm install
# 2. Run npm run build
# 3. Run npm start
```

### 5. Run Migrations

In Railway dashboard:

1. Go to **Deployments**
2. Click **Run Command**
3. Execute: `npx prisma migrate deploy`
4. Execute: `npm run db:seed` (if needed)

### 6. Custom Domain

1. Go to **Settings** → **Domains**
2. Add custom domain
3. Configure DNS records
4. Enable SSL/TLS

## Docker Deployment

### Build Docker Image

```bash
docker build -t starlabs:1.0.0 .
```

### Run Container

```bash
docker run -p 3000:3000 \
  -e DATABASE_URL="postgresql://..." \
  -e REDIS_URL="redis://..." \
  -e NODE_ENV=production \
  starlabs:1.0.0
```

### With Docker Compose (Production)

```bash
docker-compose -f docker-compose.prod.yml up -d
```

## Kubernetes Deployment

### Prerequisites
- kubectl configured
- Kubernetes cluster access
- Docker image in registry

### Create Deployment

```yaml
# k8s/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: starlabs
spec:
  replicas: 3
  selector:
    matchLabels:
      app: starlabs
  template:
    metadata:
      labels:
        app: starlabs
    spec:
      containers:
      - name: app
        image: ghcr.io/disputestrike/starlabs:latest
        ports:
        - containerPort: 3000
        env:
        - name: DATABASE_URL
          valueFrom:
            secretKeyRef:
              name: starlabs-secrets
              key: database-url
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
```

Deploy:
```bash
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
```

## Troubleshooting

### High Memory Usage
- Reduce Node pool size
- Enable garbage collection
- Check for memory leaks

### Database Timeouts
- Increase timeout values
- Check connection pool
- Optimize queries

### Slow Response Times
- Enable caching
- Optimize database queries
- Check CDN configuration
